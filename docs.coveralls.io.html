Getting Started

Coveralls is a web service to help you track your code coverage over time, and ensure that all your new code is fully covered.

There is but one prerequisite for Coveralls Cloud (Coveralls Enterprise can use a variety of repo-hosting options):

Your code must be hosted on GitHub, BitBucket, or GitLab

The Coveralls service is language-agnostic and CI-agnostic, but we haven’t yet built easy solutions for all the possibilities as far as repo hosting. Creating an account is fast and easy, just click the “Sign in” button for your git service (you can link accounts later) and authorize Coveralls to access your repos - no forms to fill out.

Using a language specific wrapper

We have everything you need ready to go if your project is:

  • Written in Ruby, Python, PHP, Node.js, C/C++, Scala, and more…
  • Building on GitHub Actions, Travis CI (open source), Travis Pro (private repos), CircleCI, Jenkins, Semaphore, Codeship, GitLab CI, and more.

If that sounds like you, take a look at our language specific implementation guides on the left-hand menu.

Sending data via the Coveralls API

If you aren’t using a CI for your project, or it isn’t using Github or Bitbucket, you can still use Coveralls through our API. We’re working to make solutions for other platforms easier going forward.
